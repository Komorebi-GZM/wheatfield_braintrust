【乡村教育 AI】用 SOLO 从零打造乡村教师专属 AI 教研搭档——麦田智囊

---

## ①摘要

面向乡村教师群体，在备课和教学经验传承场景下提供 AI 辅助能力（极速备课 + 手写教案数字化传承），通过本地 Demo 验证，可在 10 秒内生成乡土化教案、自动识别结构化老教师手写教案并入知识库。

---

## ②真实场景与需求

**目标人群**：乡村及偏远地区中小学教师

**痛点描述**：
- **经验断层**：老教师退休后，数十年积累的教学经验随人带走，新教师无人指导
- **备课负担重**：乡村教师往往身兼多科，备课时间严重不足，且现有教材/资源以城市化为导向，缺乏乡土化案例（如用"地铁"讲物理，乡村孩子根本没见过）
- **抽象概念难懂**：乡村学生缺乏生活参照物，对抽象概念理解困难

**现有做法与不足**：
- 依赖老教师"传帮带"——但老教师退休后不可持续
- 使用通用 AI 工具（如 ChatGPT）备课——生成内容缺乏乡土化，不符合乡村教学实际
- 手写教案堆在档案柜——纸质化存储，无法检索复用

---

## ③作品介绍

**麦田智囊**——基于 LangChain 与自进化 RAG 的乡村教育 Agent 系统

一个 Web 应用（Streamlit），核心包含两大 Demo 功能：

| 功能 | 说明 |
|------|------|
| **极速备课** | 教师输入课程主题（文本/语音），系统从知识库检索相关教材，10 秒内生成严格乡土化的教案（必须用乡村生活比喻，禁止城市化案例） |
| **智慧传承** | 上传老教师手写教案图片，OCR 识别 → 自动结构化整理 → 入库为知识库新内容，实现经验数字化传承 |

完整框架还包含课堂伴教、具象化素材推荐、教研纪要三大 Agent（框架已搭建，待集成）。

## ④用 SOLO 实现的过程

### 第一步：文档阅读与框架梳理

项目有三份源文档（PRD、文档合集、Demo 规格），总计数万字。我让 SOLO 逐一阅读并生成了一份《任务框架总览》，把项目定位、架构、技术栈、Demo 规格等十个模块梳理清楚，后续开发全部以此为蓝图。

> **Prompt 示例**：`/plan 阅读文档，了解任务框架`

### 第二步：任务拆解与并行开发

基于框架总览，我让 SOLO 按分层架构拆解任务，同时启动多个子 Agent 并行开发：

- **子 Agent 1**：开发计划文档 + 配置层/工具层/Prompt 层代码
- **子 Agent 2**：RAG 层 + Agent 层 + 记忆层代码
- **子 Agent 3**：补充遗漏文件

> **Prompt 示例**：`根据 demo，进一步完成开发方向和文档需求`（选择"全部都要"）

最终一次性产出 28 个文件，包含 14 个核心 Python 模块。

### 第三步：验证与修复

SOLO 自动对所有代码做了 `ast.parse()` 语法检查和导入路径验证，发现了 3 个缺失依赖（langchain-huggingface、langchain-chroma、numpy），自动修复。

### 踩过的坑

| 问题 | 原因 | 解决 |
|------|------|------|
| 中文文件名导致 Shell 命令失败 | `《"麦田智囊"》` 中的引号字符 | 改用 Python glob 处理 |
| `ModuleNotFoundError: langchain.prompts` | LangChain 升级后模块迁移到 langchain_core | 修改所有导入路径 |
| PaddleOCR 安装失败 | 不支持 Python 3.14 | 替换为 EasyOCR |
| Git 推送失败 | SOLO 运行环境的网络无法访问 GitHub | 提供本地终端命令手动推送 |

### 第四步：完整性审计

让 SOLO 对全部代码做系统审查，按 P0-P3 四级输出问题清单，明确后续开发方向。

> **Prompt 示例**：`demo还需要什么`

