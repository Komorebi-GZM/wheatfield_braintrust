# "麦田智囊"乡村教育Agent系统 — 任务框架总览

## 一、项目理解（任务重述）

### 1.1 项目全称
**"麦田智囊"：基于 LangChain 与自进化 RAG 的乡村教育 Agent 系统**

### 1.2 项目定位
基于大模型多智能体技术的**乡村教育普惠解决方案**，核心定位为**乡村教师专属AI教研搭档**。

### 1.3 核心目标
- 打造乡村教师专属的AI教研搭档
- 实现教学经验数字化传承
- 缩小城乡教育鸿沟

### 1.4 技术底座
- **核心编排框架**：LangChain（LCEL表达式语言）
- **知识库**：自进化 RAG（Chroma + BGE-Large-zh）
- **架构**：多模态 Multi-Agent 四层架构（感知层→规划层→执行层→记忆层）

## 三、系统架构总览

### 3.1 四层全链路架构

```
┌─────────────────────────────────────────────────┐
│                   感知层 (Perception)              │
│  ASR语音识别(Whisper) + OCR手写体识别(PaddleOCR)  │
│  口语化指令预处理 + 文本清洗与结构化               │
├─────────────────────────────────────────────────┤
│                   规划层 (Planning)                │
│  LangChain Router 意图路由器                      │
│  Few-Shot示例校验 + 路由结果二次校验              │
├─────────────────────────────────────────────────┤
│                   执行层 (Execution)               │
│  5大核心Agent执行链 + RAG检索引擎                 │
│  Prompt模板引擎 + 多模态素材调用工具              │
├─────────────────────────────────────────────────┤
│                   记忆层 (Memory)                  │
│  短期对话记忆 + 长期教师专属画像向量库            │
│  共享教研知识库 + 教师数字分身演进                │
└─────────────────────────────────────────────────┘
```

### 3.2 五大核心Agent链

| Agent链 | 核心功能 | 性能要求 | 核心技术 |
|---------|---------|---------|---------|
| **极速备课Agent** | 语音驱动全流程备课，生成乡土化教案包 | ≤10秒 | LangChain LCEL + RAG检索 + 风格记忆召回 |
| **智慧传承Agent** ⭐ | 手写教案OCR识别→结构化→入库→风格学习 | ≤5秒/张 | PaddleOCR + 结构化拆解Agent + 风格学习Agent |
| **课堂实时伴教Agent** | 课堂极速出题、经典题检索 | ≤3秒 | Fast_Quiz_Chain极简链路 + 纯检索模式 |
| **具象化素材Agent** | 自动匹配科普视频/3D模型/动画素材 | 联动触发 | 素材库检索 + 知识点标签匹配 |
| **教研纪要Agent** | 录音转写→结构化教研报告→成果入库 | 转写≥95%准确率 | Whisper转写 + 结构化提取Agent |

### 3.3 自进化RAG知识库（核心基建）

- **三层知识库架构**：通用基础库（教材）→ 学校共享库（校本教案）→ 教师专属库（个人风格）
- **混合检索策略**：向量检索（语义匹配）+ BM25关键词检索（专有名词精准匹配）
- **低门槛入库机制**：一键投喂、划线即入库、语音备注入库
- **知识边界强制约束**：大模型仅基于检索内容生成，禁止超纲

### 3.4 个性化记忆与教师数字分身

- **短期记忆**：ConversationBufferWindowMemory，保留最近5轮对话
- **长期记忆**：教师专属画像User Profile，风格标签向量化存储，物理隔离
- **数字分身演进**：6个月数据沉淀 → LLaMA-Factory LoRA微调 → Qwen2-7B本地部署

## 五、核心痛点与解决方案映射

| 痛点 | 解决方案 | 核心Agent |
|------|---------|----------|
| 备课需2-3小时，课件脱离乡村实际 | 语音极速备课，乡土化教案10秒生成 | 极速备课Agent |
| 抽象知识难讲解，无可视化素材 | 自动匹配科普视频/3D模型，投屏即用 | 具象化素材Agent |
| 老教师手写教案退休即流失 | OCR识别→结构化→入库→风格学习→可检索传承 | 智慧传承Agent ⭐ |
| 课堂突发场景无应急支撑 | 3秒极速出题/检索经典题 | 课堂实时伴教Agent |
| 留守儿童课后辅导断层 | 教案一键转发家长群，低门槛辅导支撑 | 家校协同功能 |
| 教研成果难以沉淀 | 录音转写→结构化报告→成果自动入库 | 教研纪要Agent |

## 九、SOLO开发过程要点

项目全程使用 TRAE SOLO 工具完成，核心利用了以下能力：
1. **Plan辅助规划** — 将Multi-Agent系统拆解为6个可并行子任务
2. **Coder智能体** — 生成LangChain Agent链代码、RAG知识库代码、Streamlit前端代码
3. **多任务并行** — 同时推进Agent链开发与UI搭建
4. **工具调度集成** — 一键调用LLM、Chroma、PaddleOCR、Whisper等
5. **上下文压缩** — 解决长代码/长文档导致的模型失焦
6. **多智能体协同** — 并行完成代码开发、文档撰写、测试设计
